# DAX Measures: Analytical Layer Design

## Overview

The analytical layer was developed in Power BI using DAX (Data Analysis Expressions) to ensure consistent, accurate, and reusable calculation of key indicators.

Given the original data quality issues (duplicates and inconsistent grain), particular care was taken to design measures that correctly reflect participant-level metrics.

---

# 1. Core Design Principle

The most critical consideration was the correct interpretation of the dataset grain:

> **One row represents a unique ParticipantID, ComponentID, and YearKey combination**

As a result:

* A participant may appear multiple times across components
* Simple row counts would lead to inflated metrics

**Therefore:**

> All participant-based indicators use `DISTINCTCOUNT`

---

# 2. Core Measures

## 2.1 Total Participants

```DAX
Total Participants =
DISTINCTCOUNT(FactParticipant[ParticipantID])
```

**Purpose:**
Counts unique participants in the programme.

**Rationale:**
Avoids double-counting due to multiple component participation.

---

## 2.2 Indirect Participants

```DAX
Indirect Participants =
SUM(FactParticipant[YouthHouseholdCount])
```

**Purpose:**
Captures additional individuals impacted through participants’ households.

**Rationale:**
Each record contributes independently to indirect reach.

---

## 2.3 Jobs Created

```DAX
Jobs Created =
SUM(FactParticipant[JobsCreated])
```

**Purpose:**
Measures employment outcomes generated by participants.

**Rationale:**
Jobs created is an additive metric across all records.

---

# 3. Disaggregated Measures

## 3.1 Female Participants

```DAX
Female Participants =
CALCULATE(
    [Total Participants],
    DimGender[GenderName] = "Female"
)
```

## 3.2 Male Participants

```DAX
Male Participants =
CALCULATE(
    [Total Participants],
    DimGender[GenderName] = "Male"
)
```

**Purpose:**
Provides gender-disaggregated analysis, a key requirement in M&E systems.

**Rationale:**
`CALCULATE` modifies filter context to isolate specific groups.

---

# 4. Outcome-Based Measures

## 4.1 Participants with Jobs

```DAX
Participants with Jobs =
CALCULATE(
    DISTINCTCOUNT(FactParticipant[ParticipantID]),
    FactParticipant[JobsCreated] > 0
)
```

**Purpose:**
Counts participants who have generated at least one job.

---

## 4.2 Participants with Enterprise

```DAX
Participants with Enterprise =
CALCULATE(
    [Total Participants],
    FactParticipant[HasEnterprise] = 1
)
```

---

---

# 5. Ratio and Performance Indicators

## 5.1 Percentage Female

```DAX
% Female =
DIVIDE(
    [Female Participants],
    [Total Participants]
)
```

---

## 5.2 Enterprise Rate

```DAX
Enterprise Rate =
DIVIDE(
    [Participants with Enterprise],
    [Total Participants]
)
```

---

## 5.3 Loan Access Rate

```DAX
Loan Access Rate =
DIVIDE(
    [Participants with Loan],
    [Total Participants]
)
```

---

## 5.4 Jobs per Participant

```DAX
Jobs per Participant =
DIVIDE(
    [Jobs Created],
    [Total Participants]
)
```

**Rationale:**
`DIVIDE` is used instead of direct division to handle division-by-zero scenarios safely.

---

# 6. Analytical Design Considerations

## 6.1 Avoiding Double Counting

* Use of `DISTINCTCOUNT` ensures participant-level accuracy
* Prevents inflation due to multiple component records

---

## 6.2 Filter Context Awareness

* Measures dynamically respond to filters (Year, Province, Component, etc.)
* `CALCULATE` enables controlled modification of context

---

## 6.3 Reusability

* Measures are defined once and reused across visuals
* Ensures consistency across the dashboard

---

# 7. Alignment with M&E Systems

The measures were designed to align with common M&E reporting requirements:

* Disaggregation (gender, geography, program component)
* Outcome tracking (jobs, enterprises, access to finance)
* Performance indicators (rates and ratios)

---

# 🎯 Conclusion

The DAX layer ensures that all reported indicators are:

* Accurate
* Consistent
* Scalable

This provides a reliable foundation for evidence-based decision-making and performance monitoring within an MIS framework.
